select_task(self, node, tasks_dict)

source code  
Returns bool, task should be executed
 * side-effect: set task.options

Tasks should be executed if they are not up-to-date.

Tasks that cointains setup-tasks must be selected twice,
so it gives chance for dependency tasks to be executed after
checking it is not up-to-date.

run_tasks(self, task_dispatcher)

source code 

This will actually run/execute the tasks. It will check file dependencies to decide if task should be executed and save info on successful runs. It also deals with output to stdout/stderr.
