  • doit.__main__

  • doit.api: Definition of stuff that can be used directly by a user in a dodo.py file.

  • doit.cmd_base
  • doit.cmd_clean
  • doit.cmd_completion: generate shell script with tab complention code for doit commands/tasks
  • doit.cmd_dumpdb

  • doit.cmd_run
  • doit.cmd_strace
  • doit.cmdparse: Parse command line options and execute it.
  • doit.compat: stuff dealing with incompatibilities between python versions
  • doit.control: Control tasks execution order
  • doit.dependency: Manage (save/check) task dependency-on-files data.
  • doit.doit_cmd: doit CLI (command line interface)
  • doit.exceptions: Handle exceptions generated from 'user' code
  • doit.filewatch: Watch for modifications of file-system use by cmd_auto module
  • doit.loader: Loads dodo file (a python module) and convert them to 'tasks'
  • doit.reporter: Reports doit execution status/results
  • doit.runner: Task runner
  • doit.task: Tasks are the main abstractions managed by doit
  • doit.tools: extra goodies to be used in dodo files
  • doit.version: doit version, defined out of __init__.py to avoid circular reference
