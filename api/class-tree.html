        • doit.tools.LongRunning: Action to handle a Long running shell process, usually a server or service.
      • doit.action.PythonAction: Python action.
        • doit.tools.PythonInteractiveAction: Action to handle Interactive python:
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.

        • doit.dependency.DatabaseException: Exception class for whatever backend exception
        • doit.exceptions.InvalidCommand: Invalid command line argument.
        • doit.exceptions.InvalidDodoFile: Invalid dodo file
        • doit.exceptions.InvalidTask: Invalid task instance.
    • doit.exceptions.CatchedException: This used to save info from catched exceptions The traceback from the original exception is saved
      • doit.exceptions.DependencyError: Error while trying to check if task is up-to-date
      • doit.exceptions.SetupError: Error while trying to execute setup object
      • doit.exceptions.TaskError: Error while trying to execute task.
      • doit.exceptions.TaskFailed: Task execution was not successful.
        • doit.exceptions.UnmetDependency: Task was not executed because a dependent task failed or is ignored
    • doit.cmdparse.CmdOption: a command line option
    • doit.cmdparse.CmdParse: Process string with command options
      • doit.cmdparse.TaskParse: Process string with command options (for tasks)
    • doit.cmd_base.Command: third-party should subclass this for commands that do no use tasks
      • doit.cmd_base.DoitCmdBase: subclass must define: cmd_options => list of option dictionary (see CmdOption) _execute => method, argument names must be option names
        • doit.cmd_auto.Auto: the main process will never load tasks, delegates execution to a forked process.
        • doit.cmd_clean.Clean
        • doit.cmd_forget.Forget
        • doit.cmd_help.Help
        • doit.cmd_ignore.Ignore
        • doit.cmd_info.Info: command doit info
        • doit.cmd_list.List
        • doit.cmd_resetdep.ResetDep
        • doit.cmd_run.Run
          • doit.cmd_strace.Strace
        • doit.cmd_completion.TabCompletion: generate scripts for tab-completion
      • doit.cmd_dumpdb.DumpDB: dump dependency DB
    • doit.reporter.ConsoleReporter: Default reporter.
      • doit.reporter.ExecutedOnlyReporter: No output for skipped (up-to-date) and group tasks
      • doit.reporter.ZeroReporter: Report only internal errors from doit
    • doit.dependency.DbmDB: Backend using a DBM file with individual values encoded in JSON
    • doit.task.DelayedLoader: contains info for delayed creation of tasks from a task-creator
    • doit.dependency.Dependency: Manage tasks dependencies
    • doit.dependency.DependencyStatus: Result object for Dependency.get_status.

    • doit.control.ExecNode: Each task will have an instace of this This used to keep track of waiting events and the generator for dep nodes
    • doit.dependency.FileChangedChecker: Base checker for dependencies, must be inherited.

      • doit.dependency.TimestampChecker: Checker that use only the timestamp.
    • doit.filewatch.FileModifyWatcher: Use inotify to watch file-system for file modifications
    • doit.runner.JobHold: Indicates there is no task ready to be executed
    • doit.runner.JobTask: Contains a Task object

    • doit.dependency.JsonDB: Backend using a single text file with JSON content
    • doit.reporter.JsonReporter: output results in JSON format
    • doit.runner.MReporter: send reported messages to master process
    • doit.plugin.PluginEntry: A Plugin entry point
    • doit.control.RegexGroup: Helper to keep track of all delayed-tasks which regexp target matches the target specified from command line.
    • doit.runner.Runner: Task runner
      • doit.runner.MRunner: MultiProcessing Runner
        • doit.runner.MThreadRunner: Parallel runner using threads
    • doit.plugin.PluginEntry.Sentinel
    • doit.dependency.SqliteDB: sqlite3 json backend
    • doit.task.Task: Task
    • doit.control.TaskControl: Manages tasks inter-relationship
    • doit.control.TaskDispatcher: Dispatch another task to be selected/executed, mostly handle with MP
    • doit.cmd_base.TaskLoader: task-loader interface responsible of creating Task objects
      • doit.cmd_base.DodoTaskLoader: default task-loader create tasks from a dodo.py file
      • doit.cmd_base.ModuleTaskLoader: load tasks from a module/dictionary containing task generators Usage: `ModuleTaskLoader(my_module)` or `ModuleTaskLoader(globals())`
    • doit.reporter.TaskResult: result object used by JsonReporter
    • doit.dependency.UptodateCalculator: Base class for 'uptodate' that need access to all tasks
      • doit.task.result_dep: check if result of the given task was modified
    • doit.action.Writer: write to many streams
    • threading._Verbose
      • threading.Thread: A class that represents a thread of control.
        • doit.runner.MThreadRunner.DaemonThread: daemon thread to make sure process is terminated if there is an uncatch exception and threads are not correctly joined.
    • doit.tools.check_timestamp_unchanged: check if timestamp of a given file/dir is unchanged since last run.
    • doit.tools.config_changed: check if passed config was modified @var config (str) or (dict)
    • dict: dict() -> new empty dictionary dict(mapping) -> new dictionary initialized from a mapping object's (key, value) pairs dict(iterable) -> new dictionary initialized as if via: d = {} for k, v in iterable: d[k] = v dict(**kwargs) -> new dictionary initialized with the name=value pairs in the keyword argument list.
      • doit.cmdparse.DefaultUpdate: A dictionary that has an "update_defaults" method where only items with default values are updated.
      • doit.plugin.PluginDict: A dict where item values *might* be a PluginEntry
    • doit.tools.timeout: add timeout to task
